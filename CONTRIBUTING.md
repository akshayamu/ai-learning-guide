# Contributing Guidelines

Thank you for your interest in contributing to the **Nonprofit AI Learning Guidance System**.

This project is built with a strong focus on **trust, transparency, and equitable access to learning**.  
Please read this document carefully before contributing.

---

## Project Values

This project prioritizes:

- Guidance over persuasion
- Explainability over optimization
- Deterministic behavior over black-box models
- Accessibility over engagement metrics
- Nonprofit intent over growth hacking

All contributions should align with these values.

---

## What This Project Is

- A learning guidance system
- Retrieval-first and explainable by design
- Focused on free and open educational resources
- Built incrementally and cautiously

---

## What This Project Is Not

- A course marketplace
- A chatbot for content generation
- An engagement-optimized product
- A black-box recommendation engine

Proposals that move the project in these directions will not be accepted.

---

## How You Can Contribute

You can help in many ways:

### 1. Data & Content
- Curating high-quality free learning resources
- Verifying difficulty levels and prerequisites
- Improving metadata quality

### 2. Engineering
- Improving retrieval quality
- Adding tests and evaluation
- Refactoring for clarity and stability

### 3. Education & Pedagogy
- Reviewing recommendations for clarity
- Suggesting better learning sequences
- Flagging misleading or overly complex resources

### 4. Accessibility (Future Phases)
- Voice / IVR design ideas
- Low-bandwidth interaction flows
- Language expansion guidance

---

## Contribution Process

1. Open an issue describing your idea or change
2. Discuss before starting major work
3. Keep changes focused and minimal
4. Avoid introducing heavy dependencies
5. Document your reasoning clearly

---

## Code Style & Principles

- Prefer simple, readable code
- Avoid unnecessary abstractions
- Keep logic deterministic
- Add comments where decisions matter

---

## Questions or Uncertainty

If you are unsure whether something fits the project:
- Open an issue
- Ask before implementing

Thoughtful discussion is encouraged.

---

Thank you for contributing responsibly.
